## 1. 🟡 新增材料:拍照快照 → 自動填營養標示/成分/過敏原

拍包裝的營養標示表 → 自動辨識 8 項營養數值、成分欄文字 → 預填表單、
自動勾選過敏原(從成分文字比對 11 類關鍵字)→ 人工確認後儲存。

- **技術方向**:辨識用 AI 視覺 API(對台灣包裝的中文直式/橫式表格,傳統 OCR 準確率差,
  AI 模型效果好很多)。走 Vercel function 轉發,金鑰放環境變數。
- **成本**:每次辨識約 NT$0.1~0.5(依模型),單人用量可忽略;開放多人後要限流。
- **照片本身**:辨識完即丟,「不」存資料庫(M0 只有 512MB;未來要存照片得用外部圖床)。
- **待拍板**:
  - 照片要不要保留當佐證?(建議:第一版不保留,辨識完即丟)
  - 手機拍照為主?(材料頁在手機上要有「＋拍照新增」入口)
- **2026-07-10 補充**:食譜 Detail 頁目前完全沒有成品照片欄位(純文字+表格),
  跟這項一樣會碰到「M0 512MB 存不存照片」的問題,建議一併拍板
  (食譜照片與材料辨識照片可以共用同一套「存不存、存哪裡」的決定)。

## 2. 🟡 帳號系統:Google 快速登入(2026-07-10 拍板政策,分階段實作中)

Google OAuth 登入;使用者可投稿個人食譜,選擇公開或私人。
**這是整個平台化的分水嶺**,動到資料架構(所有資料加 `ownerId`)、
API 權限模型(從單一密碼變成 per-user 授權)、UI(個人頁)。

- **技術方向定案(2026-07-10 二次修正):手刻 OAuth,不裝 `@auth/core`**。
  原訂 Auth.js,但發現本站前端(GitHub Pages)跟 API(Vercel)是**兩個不同網域**,
  cookie-based session 會被當第三方 cookie(Safari/Firefox 預設擋、Chrome 也在收緊),
  改成 **token 帶在網址、前端存 localStorage、之後 API 呼叫帶
  `Authorization: Bearer <token>`** 這種不依賴 cookie 的做法。既然不用 Auth.js
  的 cookie session 機制,`@auth/core` 的價值所剩不多,改手刻:
  `/api/auth/google/start`(組 Google 授權網址導過去)→
  `/api/auth/google/callback`(收 `code`→跟 Google 換 id_token→驗證→自己簽一組
  JWT,`AUTH_SECRET` 簽章)→導回前端網址帶 `#token=...`,前端讀取存
  localStorage、清網址。跟現有密碼驗證(SHA-256 比對)手刻風格一致,不加依賴。
  現有的單一編輯密碼制在帳號系統上線後轉為「站長權限」,與 per-user token 雙軌並存。
- **schema 已預留**:公開參考資料與私人營運資料欄位已分離(見 db-schema.md),
  屆時私人欄位拆出去加 `ownerId` 即可。
- **拍板結果**:
  - 只開 Google 登入?→ **是**。
  - 訪客(未登入)能看什麼?→ **全部公開資料**(食譜/材料/模具),跟現況一致,
    編輯/上傳才要登入。
  - **資料歸屬政策**:材料、模具**永遠是公開資料,沒有私人切換**(呼應「公開
    材料資料庫」是本站主力價值);**食譜可以自己選公開或私人**。你現有的
    20 道食譜、59 種材料、208 筆模具,材料/模具照舊全公開,食譜歸到你的
    站長帳號名下。
  - 你自己的 20 道食譜屆時預設公開?→ 材料/模具本來就公開;食譜的預設值留到
    第二階段實作時再定(傾向全部先設公開)。
  - 私人食譜存量限制?→ **待拍板,不擋第一階段**(先不設硬限制,正式開放
    註冊前再評估用量抓門檻)。

     `scripts/backfill-recipe-owner.mjs` 標上 `recipes.ownerId = "shccgxqp@gmail.com"`,
     先用 email 當識別值(之後要不要換成 Google `sub` 待評估,換值成本低)。
  2. ✅ **v3.10.0**:信箱+密碼註冊/登入補上,`users` collection 正式建立

     (驗證信需要驗證過的網域,`bakejojo.com` 還沒買)。側欄陽春信箱密碼測試

## 6. ✅ 網址與 API 優化:每頁專屬網址(v3.8.0,2026-07-10 完成)

需求:食譜有專屬網址(分享連結直達)。

- **定案:Hash 路由**(`react-router-dom` 的 `HashRouter`,網址長 `#/r/:id/:名稱`)。
  GitHub Pages 是純靜態站,# 後面的路徑不會送到伺服器,不用 404 轉址 hack、
  不用把主網址搬去 Vercel——今天直接上線,零部署風險。
  代價:網址多一個 `#`,SEO 掛零(hash 路由連 GET 都不會帶路徑給搜尋引擎)。
  以後要換乾淨網址(`/r/:id/:名稱`,SEO 生效),把 `HashRouter` 換成
  `BrowserRouter`(+ `basename`)即可,`react-router` 的路由定義完全不用改;
  屆時再一併評估搬去 Vercel 做預渲染(呼應「公開材料資料庫要被烘焙業者搜到」的需求)。
- **網址格式定案**:`/r/<id>/<名稱>`——id 是定位依據,名稱只是分享連結好看(裝飾用,
  讀取時完全不看),改名字不會斷連結。其他頁面:`/ings`(材料)、`/molds`(模具庫)、
  `/trash`(回收桶)、`/changelog`(更新紀錄)。
- `App.jsx` 的 `view`/`selId` 從獨立 state 改成唯一信任網址(`useLocation`/`useNavigate`
  推導),所有切頁都變成 `navigate(...)`,不再有雙重狀態源。
- 食譜 Detail 頁加「🔗 複製連結」,呼應「分享連結直達」的原始需求。

## 9. ✅ 配方縮放器 + 模具資料庫(v3.3.0,2026-07-09 完成)

兩種模式:
- **按份數縮放**:全食譜通用(餅乾/馬卡龍/手指餅乾/醬料),永遠可用。
- **按模具換算**:蛋糕/磅蛋糕/塔類,「6吋配方 → 8吋模」以容積比換算;
  **食譜必須綁定模具才開放此模式**(輸入嚴謹化的動力)。

**模具刻度已定案:幾何尺寸為準、品牌當備註(粗刻度)**——
倍率由容積(直徑²×高)決定,各品牌同尺寸差異(5~10%)小於烘焙本身容錯,
品牌精算是假精度;「拉高版」= 同直徑不同高度 = 幾何制下自然是另一筆資料。
新 collection `molds`:形狀(圓/方/長條/咕咕霍夫/塔圈…)、尺寸、高、名稱備註
(品牌寫在名稱裡當識別,計算只看尺寸)。食譜加選填 `moldId`,
編輯器用模具下拉選單取代備註裡手寫的「方形模21.5cm」(與第 5 項聯動)。
塔皮類(皮按面積、餡按容積)第一版先全配方等比+提醒。
換算結果一律附註:烤溫烤時需人工調整。

## 17. ✅ UI 專業化批次(v3.6.0,2026-07-10 完成)

跟第 16 項首頁一起做,風格才會一致(首頁用的 token、轉場、按鈕樣式最好跟站內
其他地方共用同一套)。

- **Toast 通知取代 `alert()`/`prompt()`**:全站 14 處 `alert()`
  (`Detail.jsx`/`IngredientDialog.jsx`/`MoldDialog.jsx`/`RecipeDialog.jsx`/
  `ScaleDialog.jsx`/`ShoppingDialog.jsx`/`App.jsx`)+ 登入用原生 `prompt()`
  (`App.jsx:129`),跟站內自訂 `Dialog` 風格不搭,是最明顯的業餘感來源。
- **Loading 骨架屏**:開站讀取資料時 `dataSource` 只在側欄底部顯示一行小字
  (`Sidebar.jsx`),主畫面在資料回來前是空的,沒有骨架屏或明顯提示;網路失敗時
  也只是悄悄換成小字「離線快取(唯讀)」,沒有明顯提示。
- **轉場動畫**:全站目前 0 筆 transition CSS,對話框開關、頁面切換、清單篩選都是
  瞬間切換;`Dialog.jsx` 開關、`Sidebar.jsx` 手機收合都可以加淡入/滑動。
  空清單、loading、錯誤這三種狀態應該當「設計過的元件」做(文案+圖示+明確下一步),
  不是留白或瞬間閃過。
- **對話框焦點管理**:`Dialog.jsx` 有 `role="dialog"` 但沒有 focus trap,關閉後也
  不會把焦點還給觸發按鈕。
- **共用 Chip picker 元件**:`AllergenPicker`(`IngredientDialog.jsx`)、
  Sidebar 排除過敏原、模具形狀分組 chips(`MoldsView.jsx`)是同一種視覺樣式手刻
  三次,合併成一個共用元件。
- **格式化邏輯收斂**:`fmt()` 在 `calc.js`,但 `RecipeDialog.jsx`/`ScaleDialog.jsx`
  有自己的 `toFixed`/`Math.round`,`units.js` 也沒被這些地方使用,一併收斂。
- **設計 token 補齊**:延伸現有 `paper`/`ink`/`yolk`/`warn`/`ok`(`src/index.css`)
  這組 token,補 spacing scale、陰影層級(elevation)、動態效果(motion)token,
  底子不用大動就能更精緻。
