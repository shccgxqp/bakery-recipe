# 法律合規最高準則(2026-07-09 訂)

本網站一切標示相關功能,以**《食品安全衛生管理法》為最高準則**
(全文副本:[食品安全衛生管理法.md](食品安全衛生管理法.md),
以 [全國法規資料庫現行版本](https://law.moj.gov.tw/LawClass/LawAll.aspx?pcode=L0040001) 為準)。

## 一、與本站直接相關的條文

| 條文 | 內容摘要 | 對本站的意義 |
|---|---|---|
| **第 22 條** | 包裝食品應標示品名、內容物(重量遞減排列)、淨重、食品添加物名稱、有效日期、**營養標示**等 | 標籤輸出功能的欄位清單就是照這條 |
| **第 28 條** | 標示、宣傳、廣告**不得有不實、誇張或易生誤解之情形** | 本站試算值若被直接當正式標示,誤差過大即構成違反 |
| **第 45 條** | 違反第 28 條標示不實:處**新臺幣 4 萬元以上 400 萬元以下罰鍰** | 使用者的裁罰風險,也是本站免責設計要防的連帶求償來源 |
| **第 48 條** | 違反第 22 條標示規定,經命限期改正屆期不改正:3 萬~300 萬罰鍰 | 同上 |

配套法規:食藥署「包裝食品營養標示應遵行事項」(8 項必要標示、格式、
每日參考值)及其標示值誤差允許範圍——**熱量、脂肪、飽和脂肪、反式脂肪、
碳水化合物、糖、鈉之實測值不得超過標示值之 120%;蛋白質不得低於標示值之 80%**。
烘焙失水濃縮效應會直接把「生料試算值」推出容許範圍,這是本站最大的技術性風險點。

## 二、風險模型

政府允許業者以「計算方式」產生營養標示,但**數據正確性的最終責任在業者**。
本站定位是「資料庫工具/試算平台」,要切斷的是:使用者把本站產出直接印上
商業包裝 → 被抽驗出誤差超標 → 被裁罰後回頭向本站民事求償 的連帶鏈。

平台方免責的三個要件:**充分揭露計算局限、善盡提醒義務(留下鐵證)、
不作任何合規擔保**。

## 四、免責聲明標準文字(全站統一,修改要三處同步)

1. **數據僅供參考**:本站營養標示為依材料資料理論加總之試算值,僅供研發與
   個人烘焙參考,不具法律效力。
2. **製程耗損誤差**:實際成品受製程、食材差異、水分蒸發影響,數值可能明顯誤差;
   本站不保證符合《食品安全衛生管理法》及相關遵行事項之抽驗標準。
3. **商業使用自負其責**:將本站數據用於商業販售、包裝或廣告,一切法律責任
   (含標示不實之行政裁罰)由使用者自行承擔;正式量產應送 SGS、台美檢驗等
   公正單位實際化驗。

## 五、開發鐵律

- 任何「輸出可能被印上包裝」的新功能(標示下載、完整標籤、列印),
  **必須**帶浮水印+烙印聲明+下載前同意,三者缺一不可。
- **網頁常駐顯示型輸出(2026-07-11 增,標示卡頁 `/label/:id` 適用)**:
  沒有「下載」動作可卡同意,改為三件套——
  1. **建立者同意**:產生/複製連結前一次性同意,同意時間**存伺服器**
     (`users.labelTermsAcceptedAt`,比 localStorage 更能當「善盡提醒義務」
     的證據);觀看者(客人)**不設同意牆**——觀看者不是受規範當事人,
     擋他們沒有法律意義。
  2. **頁面本體**:標示卡區塊帶淡斜向浮水印(截圖也帶著走)+頁尾常駐
     不可關閉的免責文字。
  3. **法定標示不可取代聲明**:依第 22 條,包裝標示必須印在包裝上——
     頁面與同意文字都要明示「連結/QR code 不能取代包裝上的法定標示」,
     這是切斷「把網頁當正式標示」責任鏈的關鍵。
  另:此類頁面的資料端點**只回標示需要的欄位**,成本/售價/配方克數
  不出後端(「不顯示」不等於「不傳」)。
- 免責文字只能加強、不得刪弱;措辭修改需同步更新本文件第四節。
- 註冊時強制勾選服務條款(內容即第四節),同意時間存 `users.tosAcceptedAt`
  (✅ 2026-07-11 已實作);Google 首次登入的同意在第一次產生標示卡連結時補踩。
